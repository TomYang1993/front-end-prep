# Impeccable Design Config

## Design Context

### Users
Front-end developers preparing for coding interviews (JS/TS function questions, React component questions). Primary context: focused desktop sessions at a desk with a real keyboard — solving questions in the in-browser Monaco/Sandpack workspace. Secondary context: phone browsing on the go — discovering the site, scanning the question list, reading descriptions/solutions, checking pricing. **Mobile is browse-only by decision**: marketing pages and the question list adapt fully to phones; the question detail/solving workspace prompts users toward desktop rather than cramming the editor onto a small screen.

### Brand Personality
Friendly with edge. The "WTF" / "Whack The Fullstack Interview" name has attitude — playful but competent. Personality shows up in copy and small details, not in gimmicks. The product itself stays sharp and efficient, like a tool built by developers who don't take themselves too seriously but take the craft seriously.

### Aesthetic Direction
Clean developer-tool aesthetic with an existing token system: CSS variables in `styles/globals.css` (light / dark / focus themes), blue brand (#2563eb light, #3b82f6 dark), tinted neutrals, soft shadows, 8–16px radii. Tailwind v4 with `@theme` mapping tokens to utilities. Sticky blurred header. Don't introduce new colors or fonts during responsive work — adapt layout, preserve the established look.

### Design Principles
1. **Desktop is the workbench, mobile is the storefront.** Solving happens on desktop; mobile must excel at browsing, reading, and converting — not at code editing.
2. **Adapt, don't amputate (except the editor).** Marketing pages, question list, pricing, auth, discuss threads all fully usable on phones. Only the code workspace is desktop-gated, with an honest, on-brand prompt.
3. **Use existing tokens and Tailwind utilities.** Responsive changes are `max-md:` / `sm:` / `md:` utility classes per project conventions — no new CSS files, no new design vocabulary.
4. **Sensible accessibility defaults.** 44px touch targets on mobile, 16px minimum body text on phones, respect `prefers-reduced-motion`. No formal WCAG audit required.
5. **Personality in the details.** A desktop-gate message can wink ("this editor deserves a real keyboard"); layout itself stays disciplined.
